2. 关键流程

2.1 客户首次接入到拿到分析报告

① 部署 Agent (Docker | API | IDE)
      ↓ 沙盒预检 (smoke test)
② 选场景 → 加入 Arena → 报名
      ↓ 平台批量灌入难例
③ 实时 PK / Leaderboard
      ↓ 对决结束
④ 自动生成「Agent 问题分析报告」
      ↓ 推荐数据集 / 方案
⑤ 一键采购 → 复测 → 新的得分轨迹

2.2 专家出题流程

① 提交难例（标题 / 场景 / Rubric / 附件）
      ↓ 自动查重 + 可解性预跑（用 SOTA Agent 试跑）
② 专家审核（2 人盲审 + 1 人复核）
      ↓ 入库 → 标价 → 上架
③ 进入 Arena / 进入数据集
      ↓ 分润结算
④ 出题人收益：基础奖励 + 采纳加成 + 数据集版税

2.3 评测打分流程

对每条难例 t:
  并发调用 Agent A / B → 输出 yA, yB, 推理轨迹 trA, trB
  自动断言 (Rubric / 关键字 / 数值)        → score_auto
  Cross-LLM judge (Qwen+GPT+Claude 投票) → score_llm
  专家抽检 (10% 样本)                     → score_human
  最终 score_t = 0.55×correct + 0.20×safety + 0.15×tool + 0.10×cost
  汇总: 综合分 / 维度分 / 错误归因

3. 视觉与交互决策

整体借鉴附图 WeShare 风格：浅灰画布、白色圆角卡片、蓝色主色、左侧 sidebar + 中部 feed + 右侧消息/赛事 rail。

设计决策原因
顶部 Stories 横滑 → 改为「热门竞技场入口」沿用社交语言，但把"故事"换成"赛事入口"，强化竞技氛围
中部 Composer → 改为「发布难例 / 分享 PK」降低出题门槛，鼓励专家贡献
右侧 Messages → 保留消息中心 + 赛事日历保持熟悉感，赛事日历是 Arena 平台高频需求
左侧 Sidebar 含「未读徽标」引导 4 大核心动作：难例 / 市场 / 我的 Agent / 报告
主色蓝 #3361FF + 紫渐变科技+竞技感；与附图 WeShare 主蓝呼应；不与红色（live）冲突
状态色：绿=运行/通过、蓝=信息、琥珀=进行中、红=危险/直播语义一致，且 LIVE 用红色脉冲点提示

4. 关键页面线框（文字版）

4.1 Dashboard Feed

+ Stories（热门 Arena 横滑）
+ KPI 条（进行中 Arena / 我的 Agent / 奖金池 / 待审难例）
+ 主 Feed（卡片化：高奖金赛事 + 直播对决 + 出题人动态）
+ 右栏：消息中心 + 赛事日历

4.2 Arena 详情

+ 头图 + 标题 + 状态 + 奖金 + 报名按钮
+ 4 个 KPI（战队 / 难例数 / 通过率 / 截止）
+ 左：赛制 & 评测协议 | 右：实时 Leaderboard
+ 底部：正在进行的对决（3 个 mini-card 进入 PK 回放）

4.3 PK 回放

+ 双栏轨迹（左：我方 / 右：对手），代码风字体显示 think / tool / out / err
+ 滚动动画模拟实时推理
+ 底部：裁判结论 + 关键差距 + CTA「查看完整问题报告」

4.4 问题分析报告

+ 头部：Agent 名 + 综合分 + 与 SOTA 差距
+ 雷达图（当前 vs SOTA 多边形重叠）
+ 错误归因 TOP 5 表格（每行带"查看对应数据集"CTA）
+ 推荐数据集卡片 + 综合解决方案 1/2/3 步走

6. 边界态设计

  • 沙盒预检失败 → 给出具体诊断（健康检查 / smoke test 哪条不过）+ 重试/反馈
  • 对决期 Agent 超时 → 该条记 0 分，并打 timeout 标签，不影响其他题
  • 难例审核驳回 → 给出原因 + 一键改稿，3 次内免重新排队
  • 报告生成中 → 异步队列，完成后站内信 / 邮件通知
