2. 关键组件

2.1 Sandbox Orchestrator

  • 底层：K8s + gVisor（CPU 任务） / Firecracker microVM（GPU/敏感任务）。
  • 每次评测拉起独立 Pod，含 Agent 容器 + sidecar (审计代理 / 限流 / 计时)。
  • 网络：默认 deny-all egress，按需放通的目标走平台代理（Squid + 审计）。
  • 资源配额：CPU/RAM/GPU/wall-time/输出 token 上限；超限即终止。
  • 三种接入抽象：ImageAdapter（Docker）HttpAdapter（OpenAI-兼容）IDEAdapter（平台模板）。

2.2 Eval / Judge 引擎

  • Rubric 引擎：结构化校验（JSON schema / SQL / 关键字 / 数值容差）。
  • Cross-LLM Judge：Qwen3 / GPT-5 / Claude Opus 投票，权重可配置。
  • 专家抽检：10% 样本进入 Appen 标注平台人工核验，作为长期校准。
  • 幂等 + 可重放：所有评测以 (task_id, agent_version, judge_version) 三元组哈希，结果可重算。

2.3 难例任务集 & 数据闭环

  • 难例 schema：{id, scenario, input, expected, rubric, attachments, difficulty, license}。
  • 来源：① Appen 历史标注沉淀；② 专家众包提交；③ 客户业务脱敏导入。
  • 失败样本 → 自动入"待标注队列" → 标注完成回流到对应数据集 → 数据集价格随更新自动调整。

2.4 报告生成（Insight Report）

  • 批离线 + 在线增量：完赛 30 分钟内出报告草稿；客户付费解锁完整版。
  • 结构：综合分、6 维雷达、错误归因 TOP N、推荐数据集 / MCP / 微调服务、ROI 估算。
  • 导出格式：HTML / PDF / Notion / 飞书文档。

层选型理由

BFFNode.js (Nest) 或 Go (Gin)聚合查询、WS 推送

沙盒K8s + gVisor + Firecracker强隔离 + 弹性扩缩

存储PostgreSQL + ClickHouse + S3事务 + 海量轨迹分析 + 大对象
检索OpenSearch轨迹 / 失败样本全文检索
可观测OpenTelemetry + Grafana + Loki端到端 trace + 日志
LLM 路由LiteLLM / 自研路由评测时多模型投票

6. 演进路线

阶段范围交付物
M0 · Demo（本仓）静态前端 + Mock 数据闭环可点击 demo + 4 份 HTML 文档
M1 · MVP（6 周）3 种接入 + 沙盒 + 1 个 Arena 跑通 + 报告生成邀请制内测，对 2~3 家客户付费
M2 · GA（3 个月）众包难例 + 数据集商城 + 方案市场 + 评测大盘季度活跃 ≥ 12 Arena，GMV 启动
M3（半年）多模态 Arena（视觉/语音/机器人）+ 私有化部署行业头部客户 LH

7. Demo 实现说明

本仓 demo 是纯前端 SPA（无后端依赖），通过 hash 路由切换 9 个主视图，所有数据来自 assets/app.js 中的 DB / REPORT mock。运行方式：

cd agent-arena-demo
python3 -m http.server 8080
# 浏览器打开 http://localhost:8080

覆盖的可测路径：

  • #/dashboard · #/arenas · #/arena/arena-001
  • #/agents · #/agents/new（含 4 步部署向导动画）
  • #/tasks · #/tasks/new
  • #/pk（双轨推理轨迹动画） · #/report（SVG 雷达图）
  • #/datasets · #/marketplace · #/profile
