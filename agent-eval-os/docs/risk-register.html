1. 风险总览

排行榜不可信 高 任务泄露、隐藏集污染、刷榜、样本量不足导致排名不能代表真实能力。 隐藏集隔离、赛季任务版本化、异常检测、人工抽查、置信区间、按场景分榜。 评测算法 + 运营

数据产品转化弱 中 用户看完报告但不理解为什么要采购训练数据。 失败类型到 SKU 的映射表、ROI 估算、样例数据预览、低门槛 pilot 包。 销售 + 产品

2. 隐私与数据治理要求

  • 所有真实客户数据进入任务池前必须签署用途授权，并执行脱敏、采样和审计。
  • 隐藏集任务不得向参赛 Agent 开发者暴露原始答案、verifier 细节和专家轨迹。
  • 专家用户只可访问被分配任务所需的最小信息，敏感字段默认脱敏。
  • 报告中的轨迹证据需要按客户、项目、角色做访问控制。
  • 训练数据产品交付前必须明确客户是否允许跨客户泛化使用。

3. 评测可信度控制

问题控制策略
任务太简单设定难例准入标准：多工具、长上下文、状态变化、异常返回、合规边界至少命中两项。
任务太主观主观任务采用 pairwise preference，并标注偏好理由；不与二元 verifier 混为一谈。
Agent 过拟合公开集公开集只用于调试；正式排名依赖隐藏集和赛季新题。
人工复核不一致双人标注、冲突仲裁、专家一致性评分、rubric 示例校准。
单一综合分掩盖短板必须展示分项分和场景榜，避免“总分高但不适合客户场景”。

4. 上线前 Gate

  • 完成 3 个行业场景、每个场景至少 50 个公开任务和 30 个隐藏任务。
  • 至少 2 类 Agent 接入方式通过端到端测试：API Endpoint 和容器镜像。
  • 每个场景至少有 20 条专家金标轨迹，用于报告和训练数据样例。
  • 完成沙盒安全评估、提示注入红队和隐藏集泄露演练。
  • 报告中的数据产品建议通过销售团队访谈验证，至少获得 3 个客户 pilot 意向。
