1. 产品定位

Agent Arena 是数据标注平台面向 Agent 时代的业务拓展模块。它既是客户评测与选型平台，也是训练数据需求发现系统：通过真实难例竞技识别 Agent 缺陷，再把缺陷转化为 Appen 可交付的数据产品。

对客户

评测与选型

低成本验证 Agent 在真实场景中的表现，获得可解释排名与采购建议。

对开发者

曝光与优化

通过奖金池和排行榜吸引 Agent 开发者，提供轨迹级问题定位。

对平台

数据增长

持续收集高价值难例、金标轨迹、偏好数据和失败分类。

3. 端到端主流程

1. 接入 Agent客户选择 API、镜像、托管沙盒或私有 VPC，并完成健康检查。
2. 供给难例专家提交真实任务、金标轨迹、verifier 和奖励建议。
3. 运行竞技平台在公开集和隐藏集上并行运行多个 Agent。
4. 排名发奖根据综合评分、隐藏集审计和人工复核生成排名。
5. 诊断缺陷聚合轨迹失败模式，输出根因与修复建议。
6. 数据转化推荐 Appen 数据产品与 Agent 解决方案。

5. 评分设计

综合评分建议由自动 verifier、人工 rubric、偏好对战和运营约束共同构成。不同赛季可配置权重。

维度默认权重计算方式用途
任务成功率35%程序化 verifier + 专家复核衡量最终目标是否完成。
工具调用正确率20%工具顺序、参数、状态理解、异常处理衡量 Agent 执行可靠性。
安全合规率20%高风险动作、PII、权限越界、拒绝策略适合企业客户采购决策。
成本效率10%token、工具调用、运行时长、重试次数用于商业落地评估。
用户体验/偏好15%专家 pairwise preference 或 rubric覆盖非确定性任务质量。

6. 页面级验收标准

页面验收标准
总览用户能在 30 秒内理解平台价值链：Agent、任务、竞技、报告、数据采购。
Agent 工坊用户能提交一个 mock Agent，并看到它进入已接入列表。
难例市场用户能提交一个 mock 难例，并看到它进入审核队列。
竞技运行用户能点击运行，并看到日志滚动、评测结果和失败模式。
分析报告用户能切换诊断、方案、数据、轨迹四个视角，并从报告进入采购短名单。
方案推荐用户能看到按场景推荐的 Agent 方案。
数据商店用户能把多个数据包加入短名单，并看到短名单数量变化。

7. 视觉与交互原则

  • 定位为企业运营工具，采用克制、信息密度高、可扫描的后台风格，而不是营销落地页。
  • 核心色使用深色侧栏、白色内容区、青绿色行动色，并用琥珀、红色、靛蓝承载状态含义。
  • 卡片只用于独立信息块和重复项，页面主体以表格、表单、日志和报告面板为主。
  • 所有关键流程都有即时反馈：toast、列表新增、日志滚动、短名单计数。
